keyword / tag

dcat:keyword

rdfs:Literal

This property contains a keyword or tag describing the Dataset. For the purposes of mobility data portals, it is not recommended to use keywords, as they might be ambiguous, language-dependend and make data search difficult. Instead, usage of mobility-related properties with Controlled Vocabularies is recommended, in particular “theme/ category” , "network coverage" and "transportation mode". If there are relevant keywords that are not part of such Controlled Vocabularies, please contact the authors of "mobilityDCAT-AP", in order to update the Controlled Vocabularies.

+network coverage

mobilitydcatap:networkCoverage

skos:Concept

This property describes the part of the transport network that is covered by the delivered content. For road traffic, the property is meant to refer to the network typology for which the data is provided. As a minimum, an international or higher-level classification, e.g., via functional road classes, is recommended, to allow data search across different countries. In addition, national classifications are allowed. For other transport modes, the property is meant to refer to the physical infrastructure which is used by the services covered by the data. For all modes, the property should indicate if the data content relate to the EU’s trans-European transport network (TEN-T network, https://transport.ec.europa.eu/transport-themes/infrastructure-and-investment/trans-european-transport-network-ten-t_en).

+rights holder

dct:rightsHolder

foaf:Agent

This property refers to an entity that legally owns or holds the rights about the data provided in a dataset. This entity is responsible for the content of the data, in particular, that the data fulfilles quality criteria (if applicable, see property “Quality description”) and/or is subject to legal frameworks (if applicable, see property “Legal frameworks”). The “Data Owner” will be in many cases identical with the “Publisher”. In this case, the contact data will be copied from the “Publisher” entry. There might be also cases where one or several “Data Owners” contract a third-party “Publisher” (e.g., an IT service company) to host and publish a data set. This entity is the direct contact for the platform operator or data-consuming users, who have questions or issues about the contents of a dataset, for example. The information might be identical to the property "Metadata Point of Contact" and/or “Publisher”. When provided, it should include, as a minimum, the name and email address of the entity.
