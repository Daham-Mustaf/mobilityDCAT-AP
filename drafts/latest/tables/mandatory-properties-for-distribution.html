Card.

access URL

dcat:accessURL

rdfs:Resource

This property contains a URL that gives access to a Distribution of the Dataset. Depending ont he type of data provision, there are different options for describing the URL.

For data services, e.g., API, broker services, data feeds, etc., the URL MAY be the end point of such service. However, if some agreements between the data provider and the data user need to be established first, the access URL may link to web page, where the access is further arranged, e.g., initiating a a subscription process. For data which can be downloaded directly, the download URL should be copied from the optional property "dcat:downloadURL". For other cases, where data cannot be directly accessed or downloaded via the platform, the access URL MAY be a link to an external service or a website which enables access to the data.

This way, this property is the primary key to link to a data sercice. There is also specific class "Data Service", which could be linked via property "dcat:accessService". See for a usage note about the class "Data Service".

+data model

mobilitydcatap:dataModel

skos:Concept

This property describes the data model of the delivered content within the Distribution. It desctibes the concrete data models that use the specification elements to specify specific data models. A controlled vocabulary is provided.

format

dct:format

dct:MediaTypeOrExtent

This property refers to the technical syntax of the delivered content within the Distribution. It describes the base standard that specifies syntactically correct documents. On this level, only base elements of building documents properly are specified and can be proved by syntax checks. A controlled vocabulary is provided.

The obligation of this property is raised to “mandatory”, comparing to [[DCAT-AP-v2.0.1]].

There is a similar property in [[DCAT-AP-v2.0.1]] calles "dcat:mediaType", which is removed in mobilityDCAT-AP. There reason is the use of Controlled vocabularies: "dct:format" uses the EU Authority List [[EUV-FT]], whereas "dcat:mediaType" uses the IANAmedia types [[IANA-MEDIA-TYPES]]. The first one, however, is sufficient to describe the syntaxes commonly used in mobility data portals.

rights

dct:rights

dct:RightsStatement

This property links to a Rights Statement, i.e., a statement about the intellectual property rights (IPR). As a minimum, it should include an information on the type of conditions for access and usage. A concrete (standard) license can be referred via property "dct:license.

The obligation of this property is raised to “mandatory”, comparing to [[DCAT-AP-v2.0.1]].
