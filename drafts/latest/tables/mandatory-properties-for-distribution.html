Card.

access URL

dcat:accessURL

rdfs:Resource

This property contains a URL that gives access to a Distribution of the Dataset. Depending ont he type of data provision, there are different options for describing the URL. For data services, e.g., data feeds, the URL might be the API access. However, if some agreements between the data provider and the data user need to be established first, the access URL becomes no public metadata, but is linked to a subscription that enables the access to the publication. For data which can be downloaded directly, the download URL should be copied from the optional property "donwload URL". For other cases where data cannot be directly accessed or downloaded via the platform, the ULR is a link to a service or a website which enables access to the data, e.g., a website where a data feed or a data download can be applied for.

This property refers to the technical syntax of the delivered content within the Distribution. It describes the base standard that specifies syntactically correct documents. On this level, only base elements of building documents properly are specified and can be proved by syntax checks.

rights

dct:rights

dct:RightsStatement

This property indicates the conditions if any contracts, licences and/or are applied for the use of the dataset The conditions are declared on an aggregated level: whether a free and unrestricted use is possible, a contract has to be concluded and/or a licence has to be agreed on to use a dataset. A controlled vocabular is provided. The values can be logically mapped with the property “Standard licences”.
